# [TRD] 썬텍 홈페이지 개선: 기술 아키텍처 및 구현 명세서

## 1. 시스템 아키텍처 오버뷰
본 프로젝트는 **Severless & Direct-to-CDN** 아키텍처를 채택하여 전 세계 어디서나 최상의 속도로 썬텍의 대용량 미디어 데이터를 서빙하고, 1인 운영 환경에서도 무중단 운영이 가능한 구조를 지향합니다.

## 2. 권장 기술 스택

### 2.1 프론트엔드 (Frontend)
- **기술:** **Next.js 14+ (App Router)**
- **이유:** 서버 사이드 렌더링(SSR)을 통한 SEO 최적화와 이미지 최적화(Next/Image)를 통해 글로벌 연구원들에게 빠른 첫 로딩 속도 제공.
- **스타일링:** **Vanilla CSS / CSS Modules 기반**
- **이유:** 디자인 시스템의 완벽한 제어와 최상의 성능 확보.

### 2.2 백엔드 및 데이터베이스 (Backend & DB)
- **기술:** **Supabase (PostgreSQL & Netlify Functions)**
- **이유:** 별도의 서버 관리 없이 인증, DB, 스토리지(파일 저장), 엣지 함수를 한 번에 해결 가능한 'Backend-as-a-Service'. AI 코딩 도구와의 호환성이 매우 우수함.
- **데이터베이스:** PostgreSQL (강력한 정형 데이터 처리 및 미래 확장성).

### 2.3 스토리지 및 자산 (Storage)
- **기술:** **Supabase Storage + CDN**
- **이유:** 고해상도 제품 이미지, 도면, 제조 설비 영상을 전 세계 엣지 노드에서 빠르게 서빙.

### 2.4 호스팅 및 배포 (Deployment)
- **플랫폼:** **Vercel**
- **이유:** GitHub 연동을 통한 자동 배포 파이프라인(CI/CD) 구축. 1인 운영에 최적화된 자동화 환경 제공.

## 3. 데이터 요구사항 및 스키마 원칙
- **정형 데이터:** 제품 스펙, 기술 특허, 회사 연혁, 문의 내역.
- **비정형 데이터:** 고해상도 제품 이미지, 공장 소개 영상(mp4), 기술 도면.
- **데이터 생명주기:** 개인정보(문의 내역)는 수집 목적 달성 후 3년 내 파기 원칙.

## 4. 비기능적 요구사항
- **성능:** LCP(Largest Contentful Paint) 2.5초 이내 달성 (글로벌 테스트 환경 기준).
- **보안:** HTTPS 필수 적용, Supabase RLS(Row Level Security)를 통한 안전한 데이터 보호.
- **접근성:** 다크 모드 고대비 가이드 준수 (WCAG 2.1 AA 기준).
- **글로벌화:** i18next 기반의 동적 다국어 처리 구조.

## 5. 예상 비용 및 확장 전략
- **초기 비용:** 무료 티어(Free Tier) 내에서 시작 가능 (Next.js + Supabase + Vercel).
- **확장 시점:** 트래픽 증가 및 대용량 파일 저장 용량 초과 시 유료 티어 전환.
- **벤더 락인 대책:** 모든 코드는 표준 SQL 및 React 기반으로 유지하여 필요 시 다른 클라우드 환경으로 이전 용이성 확보.
